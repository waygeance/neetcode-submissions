## 🧠 Core Revision Guide
*Use this section to quickly jump to specific patterns during interview prep.*

| Category | Core Techniques | Key Intuition |
| :--- | :--- | :--- |
| **Arrays & Hashing** | HashMaps, Sets, Prefix Sum | Trading space for $O(1)$ lookups. |
| **Two Pointers** | Opposite Ends, Fast/Slow | Reducing $O(n^2)$ to $O(n)$ in sorted/linked structures. |
| **Sliding Window** | Variable vs. Fixed Size | Optimizing contiguous subarray/substring problems. |
| **Backtracking** | State-Space Trees, Pruning | Systematic exploration of all permutations/combinations. |
| **Trees & Graphs** | BFS, DFS, Dijkstra, Union-Find | Mapping relationships and shortest paths. |
| **Dynamic Programming** | Memoization, Tabulation | Breaking down overlapping subproblems. |

## 🛠️ Tech Stack & Methodology
- **Primary Language:** C++ (Selected for its STL efficiency and low-level memory control).
- **Environment:** NeetCode.io integrated with GitHub Sync.
- **Commit Strategy:** Automated sync of accepted solutions with a focus on clean, readable code.

### Performance Targets
- **Time Complexity:** Aiming for the theoretical lower bound (usually $O(N)$ or $O(\log N)$).
- **Space Complexity:** Prioritizing in-place algorithms ($O(1)$) unless trade-offs are necessary for speed.
